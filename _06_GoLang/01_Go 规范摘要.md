## Go 规范摘要

### 1. 基本概念

#### 1.1. 程序结构

> **层次结构**

Go 作用域分为 *universe*（全局）、*package*（包）、*file*（文件）、*function*（函数）和 *block*（块）。大写标识符声明为外部链接。

```go
// 埃拉托斯特尼筛法, 用于生成素数
package main // 包声明
import "fmt" // 包导入

func generate(ch chan<- int) {
	for i := 2; true; i++ {
		ch <- i // Send 2,3,4,...
	}
}
func filter(src <-chan int, dst chan<- int, prime int) {
	for i := range src {
		if i%prime != 0 {
			dst <- i
		}
	}
}
func Sieve(n int) {  // 外部链接
	ch := make(chan int) 
	go generate(ch)     
	for range n {
		prime := <-ch
		ch1 := make(chan int)
		go filter(ch, ch1, prime) 
		ch = ch1
	}
}
func main() { // Main Entrypoint.
	Sieve(100)
}
```

> **程序初始化**

包初始化 ➡ 变量初始化 ➡ 按序调用全部 `func init()` ➡ 运行 `main()`。全局变量初始化或零初始化，指针、函数、接口、切片、通道和映射默认值为 `nil`。

`init` 可以启动其他 *goroutine*，可以声明任意数量的 `init`。Go 程序通过将主包（`package main`）和所有依赖包递归链接起来创建的。

```go
package main // 包声明
import "fmt" // 包导入
/* 首先初始化全局变量
   按序调用 init 函数
   最后运行 main()
*/
var V int = Init(10)    		
func main() {   // 程序入口
	println("Enter main")
}
func init() { fmt.Println("Enter init A") }
func init() { fmt.Println("Enter init B") }
func Init[T any](t T) (rt T) {
	rt = t
	fmt.Println("Init V = ", rt)
	return
}
```

> *编译与链接*

Go 编译过程：源代码（\*.go） ➡ 编译器 ➡ 目标代码（\*.o） ➡ 链接器 ➡ 可执行文件

```powershell
# 1. 编译并链接
go build main.go
# 2. 仅编译，不链接
go tool compile -o main.o main.go
# 3. 运行程序
go run main.go
# 4. 安装到 $GOPATH/bin
go install
```

```go
// 十进制
0., 72.40, 072.40, 2.71828
```


> **字符文本**

Go 使用 UTF-8 编码的 Unicode 字符，支持多字节。支持十六进制 `\x hh`、`\u hhhh`、`\U HHHHHHHH` 和八进制 `\ nnn`。

```go
'a', 'ä', '本', '\t',

// 转义字符
'\a', '\b', '\f', '\n', '\r', '\t', '\v', '\\', '\'', '\"'
```

> **字符串文本**

字符串 `string` 不可变且不可寻址，索引字符串返回字节 `byte`，迭代字符串返回字符 `rune`。`+` 拼接字符串。

```go
var str string = "Hello 世界"
println(str[len(str)-1]) // last byte = 140
for _, r := range str {  // 遍历字符
	fmt.Print(string(r) + ",") // H,e,l,l,o, ,世,界,
}
str += " and good bye" // 拼接
```

字符串支持原始字符串和解释字符串。

```go
// 原始字符串
`abc`                // "abc"
`\n
\n`                  // "\\n\n\\n"
// 解释字符串
"Hello, world!\n"
"日本語"
"\u65e5本\U00008a9e"
```

#### 2.2. 常量与变量

> **常量**

非类型化常量的默认类型是 `bool`、`rune`、`int`、`float64`、`complex128` 或 `string`。

```go
const (
	Pi float64 = 3.14159265358979323846
	zero = 0.0           // untyped, float64
	size int64 = 1024	 // typed
	eof        = -1      // untyped, int
)
const a, b, c = 3, 4, "foo"  // a = 3, b = 4, c = "foo"
const u, v float32 = 0, 3    // u = 0.0, v = 3.0
```

标识符 `iota` 在一组 `const` 声明中表示从零开始的连续整数常量，可以借助 `iota` 构建枚举常量。

```go
const (
	a  = 1 << iota //  1 << 0 (iota = 0)
	a1 = 2         //  (iota == 1, unused)
	b  = 2 * iota  //  2 * 2  (iota = 2)
	b1             //  2 * 3  (iota = 3)
	b2             //  2 * 4  (iota = 4)
	b3             //  2 * 5  (iota = 5)
	c  = 1 + iota  //  1 + 6  (iota = 6)
	c1             //  1 + 7  (iota = 7)
	c2             //  1 + 8  (iota = 8)
)
```

> **变量**

变量保存值的存储位置和数据，`_` 表示弃元。块范围内支持短变量声明：`idList := exprList`；短变量重声明要求左侧至少有一个新变量声明。

```go
var (
	U, V, W int      // 多变量声明
	i       int
	u, v, s = 2.0, 3.0, "bar"
)
func Values() {
	i, j := 0, 10
	re, im := complexSqrt(-1)
	_, y, _ := coord(p) // 弃元
	// 短变量重声明 要求左侧至少有一个新声明非空变量
	i, k := 10, j
}
```

#### 2.4. Slice 切片

切片是其底层数组某个连续段的编号序列，实现包含一个指向底层数组的指针、长度和容量。切片由切片表达式或 `make` 构造。

```go
a := [100]int{}  // arr
sa := a[10:50]   // 切片表达式，截取数组 10 到 49 元素

// make 构造
var s = make([]T, len [, cap]?)   // cap >= len
// 相当于
a := [cap]T{}
s := a[0:len]
```

切片表达式（`a[low : high <: max>?]`）从数组、数组的指针、切片中构造切片；`a[low : high]` 从字符串中构造子字符串。

```go
a := [5]int{1, 2, 3, 4, 5}
s := a[1:4]   // 索引 1~3
a[2:]  		  // a[2 : len(a)]
a[:3]         // a[0 : 3]
a[:]   		  // a[0 : len(a)]

str := "Hello"[:3]   // "Hel"
```

切片始终一维，可以组合成多维切片，每层切片单独初始化。

```go
type LinesOfText [][]byte     
text := LinesOfText{          // 每个内部切片可能具有不同的长度
    []byte("Now is the time"),
    []byte("for all good gophers"),
    []byte("to bring some fun to the party."),
}
```

#### 2.5. Map 映射

映射是一种由键值对组成的无序组。键可以是任何 `comparable` 类型（非映射、切片、函数）。映射没有定义 `==` 运算。


```go
var timeZone = map[string]int{    // 初始化器
    "UTC":  0*60*60,
    "EST": -5*60*60,
}
var m = make(map[string]int)     // make 构造
// 添加元素
m["A"] = 65
// 删除或清空
delete(m, "A")  
clear(m)
```

访问不存在 *key* 返回 *value* 类型零值。多重赋值用于检查映射是否包含该键。

```go
if value, ok := m[key]; ok{
	 // read successful
	 // and use rt ... 
}
```

> **并发中的映射**

并发中使用映射并不安全。可以对映射的访问执行同步，例如同步锁 `sync.RWMutex`。

```go
var counter = struct{
    sync.RWMutex
    m map[string]int
}{m: make(map[string]int)}

// 读取 counter，使用读锁：
counter.RLock()
n := counter.m["key"]
counter.RUnlock()

// 写入 counter，使用写锁：
counter.Lock()
counter.m["key"] = value
counter.Unlock()
```

#### 2.6. Struct 结构

结构包含一组字段成员，字段可以显式声明或是嵌入声明。嵌入字段指定为类型 `T` 或非接口 `*T`。

```go
type S struct {
	// 嵌入字段
	T1        // S.T1
	*T2       // S.T2
	P.T3      // S.T3
	*P.T4     // S.T4
	_    [4]byte     // _ 充当填充
	x, y float64 "any string is permitted as a tag"    // 字段属性，通过反射访问
	name string  `protobuf:"1"`  // key:value
}
```

嵌入字段 `S.T` 的成员或方法 `T.f` 将作为提升字段或提升方法。若 `T` 实现了接口 `I`，则 `S` 也实现了 `I`。
- `S` 嵌入 `T`，则 `S` 和 `*S` 包含 `T` 的提升方法。`*S` 还包括 `*T` 的提升方法。
- `S` 嵌入 `*T`，则 `S` 和 `*S` 包含 `*T` 或 `T` 的提升方法。

```go
type I interface {
	Func()
}
type Embed struct{}
func (Embed) Func() {}   // Embed 实现 I
type S struct {          // S 实现 I
	Embed	
}
func main() {
	s := S{}
	s.Func()     // 提升方法
	I(s).Func()     
}
```

#### 2.7. Function 函数

函数参数名称可省略。函数原型声明可以表示外部方法，例如汇编例程。

```go
func Fun( <param-list>? ) <ret-list>? { statements }
func GeneFun[ TypeParam-List ] ( <param-list>? ) <ret-list>? { statements }   // 泛型

func foo()
func foo(a, _ int, z float32) bool
func foo[T ~int](a, b T) (bool)
func foo(a, b int, z float64, opt ...any) (success bool)    
func foo(prefix string, values ...int)
func foo(int, int, float64) (float64, *[]int)
func foo() func(int,int)     // return type func(int,int)
```

函数可以直接返回参数、局部变量或复合文本的地址。

```go
func F(a int) (*int, *(struct{ x, y int })) {
	return &a, &struct{x int; y int}{ 1, 2 }
}
```

> **可变参数**

可变参函数 `func f(arg T1, p ...T)` 的可变参数 `p ...T` 等价于 `[]T`。可以传递切片 `slice...` 作为 `p` 的实参。

```go
func Greeting(prefix string, who ...string)

Greeting("nobody")   // who = nil
Greeting("hello:", "Joe", "Anna", "Eileen")  // who = []string{"Joe", "Anna", "Eileen"}

s := []string{"James", "Jasmine"}
Greeting("goodbye:", s...)   // who = s
```

> **匿名函数**

匿名函数可以作为函数文本赋值给函数类型的变量。

```go
var SumInt = func(x, y int) int { return x + y }
sum := SumInt(1, 1)
sum = func(x, y int) int { return x + y }(1, 1)
```

> **成员方法**

方法声明绑定到以 `T` 或 `T*` 为接收器的方法集。`T` 的方法集由以接收器 `T` 方法组成。`*T` 的方法集由以接收器 `*T` 或 `T` 方法组成。接口方法集是接口类型集各个类型的方法集的交集。

```go
func (t T) Method( params ) 	// receiver is T
func (pt *T) Method( params )	// receiver is *T
// 泛型
type Pair[U T1, V T2] struct {  }
func (p Pair[U, V]) Swap() Pair[V, U]  { statements }  
```

> *方法表达式*

类型 `T` 具有方法：`(T).Mv(params)` 和 `(*T).Mp(params)`。调用方法表达式 `T.Mv`,`(*T).Mp`,`(*T).Mv` 等效于 `T.Mv(t,params)`,`(*T).Mp(&t,params)`,`(*T).Mv(&t,params)`：

```go
func (tv T) Mv(int) int      { return 0 } // value receiver
func (tp *T) Mp(int) float32 { return 1 } // pointer receiver

T.Mv(T{}, 7)
(*T).Mp(&T{}, 7)
(*T).Mv(&T{}, 7)
```

#### 2.8. Interface 接口

接口定义类型集和方法集协议。接口元素可以是方法、嵌入接口或类型约束。

> **基接口**

基接口完全由方法列表组成定义。空接口 `type any = interface{}` 是 Go 统一类型系统的最终基。

```go
type File interface {
	Read([]byte) (int, error)
	Write([]byte) (int, error)
	Close() error
}
```

> **嵌入接口**

```go
type Reader interface {
	Read(p []byte) (n int, err error)
	Close() error
}
type Writer interface {
	Write(p []byte) (n int, err error)
	Close() error
}

// 嵌入接口，包含 Read, Write, Close 方法
type ReadWriter interface {
	Reader 
	Writer 
}
```

> **类型约束**

类型约束限定接口的类型集。类型约束项可以是任意类型 `T`（接口不包含方法）、非接口底层类型 `~T`、类型项联合 `T1 | T2 | ...`。联合约束集项之间不存在交集。

```go
interface { ~int }           // underlying type int
interface { int | string }   // type union
interface{ comparable }		 // all comparable types
interface {		// ~int that implement the String
	~int	   
	String() string
}
interface {     // 空类型集
	int			
	string
}
```

> **接口实现**

如果 `T` 实现了接口 `I` 的所有方法列表并满足类型约束，则 `T` 实现了接口 `I`。

```go
type ISample interface {
	~int | ~float32 | ~float64
	Foo()
}
type MInt int
func (MInt) Foo() { ... }    // MInt 实现了 ISample[int]
type MFloat float32
func (MFloat) Foo() { ... }  // MFloat 实现了 ISample[float32]

func Foo[I ISample](i I) {
	i.Foo()
}
func main() {
	Foo[MInt](10)
	Foo[MFloat](3.1415)
}
```

#### 2.9. Channel 通道

通道提供 *goroutine* 之间的通信机制，`<-` 指定通道方向（仅发送 `<-chan`、仅接收 `chan<-`、双通道 `chan`）。通道可以是带缓冲或无缓冲。

```go
ch := make(chan int)       // 无缓冲通道
ch := make(chan int, 10)   // 带缓冲通道

ch <- 1      // send data
rt := <- ch  // receive data
```

`close` 关闭发送通道。接收时的多重赋值用于检查在通道关闭前是否发送了数据。

```go
if _, ok := <-ch; ok {
	close(ch)
} 
```

#### 2.10. Pointer 指针

指针指向给定类型变量的内存地址，函数 `new` 返回指向给定类型零初始化的对象或表达式结果的指针。Go 指针不支持指针运算。无法 `&` 取址函数标识或匿名函数。

```go
var p1 *Point = &Point{X: 1, Y: 2}
p2 := new(Point) // *p2 = Point{X: 0, Y: 0}
p3 := new(Point{100, 200})
pm := new(map[int]bool) // nil

var pa1 *[10]int       // nil
var pa2 = new([10]int) // *pa = [10]int{}
```

可以指针转化为 `unsafe.Pointer` 或 `uintptr` 进行算术运算。

```go
arr := [3]int{1, 2, 3}
addr := uintptr(unsafe.Pointer(&arr[0]))
// 移动指针指向下一个元素
addr += unsafe.Sizeof(arr[0])
// 将 uintptr 转回 unsafe.Pointer，再转回普通指针
next := (*int)(unsafe.Pointer(addr))
fmt.Println(*next) // 2
```

#### 2.11. 命名类型

命名类型包括 *别名声明* 和 *类型定义*。类型别名与原类型等价；类型定义定义新类型，底层类型相同。

```go
type byte = uint8			// 别名
type myString string		// 新类型

var myStr myString = "Hello"
var b byte = myStr[0]
```

#### 2.12. 类型和值的属性

> **底层类型**

每个类型 `T` 都有一个底层类型。

```go
type (
	A1 = string     // string
	A2 = A1		    // string
	A3 []A1         // []A1
	A4 A3 			// []A1
)
func f[P any](x P) { … }     // P, interface{}
```

> **核心类型**

每个非接口类型 `T` 都有一个核心类型，该类型与底层类型 `T` 相同。核心类型不是命名类型、类型参数或接口类型。具有核心类型的接口示例：

```go
type Celsius float32
type Kelvin  float32
type myString string

interface{ int }                          // int
interface{ Celsius | Kelvin }             // float32
interface{ ~chan int }                    // chan int
interface{ ~chan int | ~chan<- int }      // chan<- int
interface{ ~[]*data; String() string }    // []*data
interface{ []byte | string }              // bytestring
interface{ ~[]byte | myString }           // bytestring

// 不带核心类型的接口示例：
interface{}                               // no single underlying type
interface{ Celsius | float64 }            // no single underlying type
interface{ chan int | chan<- string }     // channels have different element types
interface{ <-chan int | chan<- int }      // directional channels have different directions
```

#### 2.13. 泛型声明

通过指定类型参数列表及其约束声明，构造泛型类型或泛型方法。

```go
// 泛型声明
type T[P *C] …
type T[P (C)] …
type T[P *C|Q] …
func F[X T]( params ) { … }

// 泛型类型
type Stack[T any] struct {
	items []T
}
func (s *Stack[T]) Push(item T) {
	s.items = append(s.items, item)
}

// 泛型函数
func Min[T cmp.Ordered](a,b T) T {
	if a < b {
		return a
	}
	return b
}
```

泛型约束是定义所有允许的类型参数集的接口类型。形式为 `P E` 的泛型约束，实际上是 `P interface{E}`。

```go
[T []P]                      // 相当于 [T interface{ []P }]
[T ~int]                     // 相当于 [T interface{ ~int }]
[T int|string]               // 相当于 [T interface{ int | string }]
```

自 Go1.26 允许泛型自引用：

```go
type List[T any] struct {
	Value T
	Next  *List[T]
}

type Tree[T comparable] struct {
	Value T
	Left  *Tree[T]
	Right *Tree[T]
}

type Element[E Element[E]] interface {
	Less(other E) bool
}
type MyInt int
func (i MyInt) Less(other MyInt) bool {
	return i < other
}
```

#### 4.6. 延迟语言：Defer

`defer` 挂起一个延迟函数，参数在挂起时计算。所有延迟函数在函数返回前或发生异常时，按挂起逆序立即被调用。

```go
func PrintSlice[T any](arr []T) {
	var l sync.Mutex
	l.Lock()
	defer l.Unlock() // 返回前解锁
	for _, v := range arr {
		fmt.Printf("%v ", v)
	}
}
PrintSlice([]byte("Hello World"))
```

`defer` 常应用于解锁互斥锁、关闭文件流、处理 *panicking*。

```go
// 返回文件的正文字符串
func Contents(filename string) (string, error) {
    f, err := os.Open(filename)
    if err != nil {
        return "", err
    }
    defer f.Close() 

    var result []byte
    buf := make([]byte, 100)
    for {
        n, err := f.Read(buf[0:])
        result = append(result, buf[0:n]...) 
        if err != nil {
            if err == io.EOF {
                break
            }
            return "", err  
        }
    }
    return string(result), nil // 文件流 f 在函数返回后关闭
}
```

#### 4.7. panic、recover：异常处理

```go
func panic(v any)
func recover() any
```

函数 `panic` 和 `recover` 有助于报告和处理运行时异常和程序定义的错误条件。发生系统性异常或显式调用 `panic` 函数时， *panicking* 沿堆栈展开向上传递，直到终止程序运行或被 `recover` 捕获。程序终止时报告错误条件，并返回 `panic` 的参数值。

函数 `recover` 在 `defer` 延迟函数中执行异常处理。捕获一个 *panicking* 时，`recover` 返回调用 `panic` 传递的参数并停止 *panicking* 并恢复当前 *goroutine* 继续执行。未捕捉到 *panicking* 时返回 `nil`。

```go
func main() {
	protect(func() {
		var a, b int = 1, 0
		_ = a / b
	})
}
func protect(g func()) {
	defer func() {
		log.Println("done") 
		if x := recover(); x != nil {     // 异常处理
			log.Printf("run time panic: %v", x)
		}
	}()
	log.Println("start")
	g()
}
```

#### 5.1. Panic 和 Recover

向调用方报告错误的一种方式是额外返回一个 `error`。函数 `panic` 会创建一个运行时 *panicking*，异常传递给主例程时程序停止。函数 `recover` 用于捕获 *panicking* 以停止异常传递，通常在 `defer` 中调用。`recover` 返回 `nil` 或传递给 `panic` 的参数。

未捕获异常的 `recover` 总是返回 `nil`。可以用这类操作简化复杂软件中的错误处理，例如一个理想化的 `regexp` 包，它通过使用本地 `Error` 类型调用 `panic` 来报告解析错误。下面是 `Error` 的定义、`error` 方法和 `Compile` 函数。

```go
func Abort() {
	panic("ABORT")
}

func main() {
	defer func() {
		if r := recover(); r != nil {
			fmt.Println("Recovered:", r)
		}
	}()
	Abort()
	print("cannot arrive")
}
```

| `GOPATH`      | 在 `GOPATH` 模式下，`GOPATH` 变量是可能包含 Go 代码的目录列表。在模块感知模式下，模块缓存存储在第一个 `GOPATH` 目录的 pkg/mod 目录中。 |
| `GOPRIVATE`   | 应该被视为私有的模块路径前缀的 glob 模式的逗号分隔列表。`GOPRIVATE` 是 `GONOPROXY` 和 `GONOSUMDB` 的默认值。                           |
| `GOPROXY`     | `GOPROXY` 默认为 `https://proxy.golang.org,direct`。制定模块代理。                                                                     |
| `GOSUMDB`     | 指定要使用的校验和数据库的名称，以及可选的公钥和 URL。                                                                                 |
| `GOVCS`       | 控制 Go 命令可以用来下载公共和私有模块或匹配 glob 模式的其他模块的版本控制工具集。                                                     |
| `GOWORK`      | `GOWORK` 环境变量指示 Go 命令进入工作区模式，使用提供的 go.work 文件定义工作区。                                                       |

### 附录

#### `go test` flags

`go test` 命令同时接受应用于 `go test` 本身的标志和应用于结果测试二进制文件的标志<sup>[[↗]](https://pkg.go.dev/cmd/go#hdr-Testing_flags)</sup>。有几个标志控制分析并编写适合 `go tool pprof` 的执行配置文件。

| flag                          | description                                                                                                                                                                               |
| :---------------------------- | :---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `-bench regexp`               | 运行匹配正则的基准测试，反斜杠（`/`）用于分割元素的测试（例如子测试）                                                                                                                     |
| `-benchtime t`                | 运行每个基准测试足够迭代的时长（`time.Duration`，默认 1s）或运行次数（`Nx`）                                                                                                              |
| `-count n`                    | 运行每个 test，benchmark，fuzz 的次数，默认 1                                                                                                                                             |
| `-cover`                      | 启用覆盖率分析                                                                                                                                                                            |
| `-covermode set,count,atomic` | 为测试的包设置覆盖率分析模式                                                                                                                                                              |
| `-coverpkg pattern1,pattern2` | 在每个测试中对匹配模式的包应用覆盖率分析                                                                                                                                                  |
| `-cpu 1,2,4`                  | 指定应该为其执行测试、基准测试或模糊测试的 GOMAXPROCS 值列表。`-cpu` 不适用于`-fuzz` 匹配的模糊测试                                                                                       |
| `-failfast`                   | 首次测试失败时，不执行新的测试                                                                                                                                                            |
| `-fullpath`                   | 在错误信息中显示完整文件名称                                                                                                                                                              |
| `-fuzz regexp`                | 运行匹配正则的模糊测试                                                                                                                                                                    |
| `-fuzztime t`                 | 运行每个模糊测试足够迭代的时长（`time.Duration`）或运行次数（`Nx`）                                                                                                                       |
| `-fuzzminimizetime t`         | 运行每个模糊测试足够迭代的最低限度的时长（`time.Duration`）或运行次数（`Nx`）                                                                                                             |
| `-json`                       | 以 JSON 形式记录详细输出和测试结果                                                                                                                                                        |
| `-list regexp`                | 列出匹配正则的 TestXxx、BenchmarkXxx、FuzzXxx 或 ExampleXxx，不会运行任何测试                                                                                                             |
| `-parallel n`                 | 允许并行执行调用 `t.Parallel` 测试函数，和运行种子语料库时调用 `t.Parallel` 的 fuzz targets。在进行 fuzzing 时，`n` 可以是同时调用模糊函数的子进程的最大数目（不管是否调用 `t.Parallel`） |
| `-run regexp`                 | 仅进行与正则匹配的 Test、Example、Fuzz 的测试函数                                                                                                                                         |
| `-short`                      | 告知长时间运行的测试缩短其运行时间                                                                                                                                                        |
| `-shuffle off,on,N`           | 随机化 Tests 和 Benchmarks 的执行顺序                                                                                                                                                     |
| `-skip regexp`                | 仅运行与正则不匹配的 Test、Example、Fuzz、Benchmark 测试函数                                                                                                                              |
| `-timeout d`                  | 测试超时时 *panic*，`d=0` 禁用超时                                                                                                                                                        |
| `-v`                          | 详细输出，在运行时记录所有测试，并打印 `Log` 和 `Logf` 调用中的所有文本                                                                                                                   |
| `-vet list`                   | 在 `go test` 期间配置 `go vet` 的调用                                                                                                                                                     |

以下标志用于在 `go test` 执行期间 *profile* 测试。

| flag                      | description                                                                        |
| :------------------------ | :--------------------------------------------------------------------------------- |
| `-benchmem`               | 打印基准测试时的内存分配统计信息                                                   |
| `-blockprofile block.out` | 当所有测试完成时，将运行 *goroutine* 阻塞 *profile* 写入指定文件                   |
| `-blockprofilerate n`     | 通过调用 `runtime.SetBlockProfileRate` 控制运行例程阻塞 *profile* 中提供的详细信息 |
| `-coverprofile cover.out` | 当所有测试通过后，将测试覆盖率 *profile* 写入指定文件                              |
| `-cpuprofile cpu.out`     | 退出前将 CPU *profile* 写入指定文件                                                |
| `-memprofile mem.out`     | 在所有测试通过后，将内存分配 *profile* 写入指定文件                                |
| `-memprofilerate n`       | 通过设置 `runtime.MemProfileRate` 来启用更精确（和昂贵）的内存分配 *profile* 文件  |
| `-mutexprofile mutex.out` | 当所有测试完成后，将互斥锁争用 *profile* 写入指定文件                              |
| `-mutexprofilefraction n` | 采样 goroutine 中持有争用互斥锁的 1/n 堆栈跟踪                                     |
| `-outputdir directory`    | 将概要 *profile* 的输出文件放在指定目录                                            |
| `-trace trace.out`        | 退出前将执行的追踪写入指定文件                                                     |
